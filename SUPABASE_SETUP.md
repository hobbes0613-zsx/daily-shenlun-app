## 第一步：创建数据库表（2分钟）

### 1.1 打开 SQL Editor
1. 在Supabase项目首页，点击左侧菜单的 **SQL Editor**
2. 点击 **New Query** 按钮

### 1.2 复制并执行SQL脚本

```sql
-- 创建 daily_news 表
CREATE TABLE IF NOT EXISTS daily_news (
  id INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  date TEXT NOT NULL,
  data JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建唯一索引（防止同一天重复数据）
CREATE UNIQUE INDEX IF NOT EXISTS daily_news_date_key 
ON daily_news(date);

-- 创建更新时间触发器
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_daily_news_updated_at 
BEFORE UPDATE ON daily_news 
FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

-- 插入测试数据
INSERT INTO daily_news (date, data) 
VALUES (
  '2025-01-18',
  '[
    {
      "id": 1,
      "rank": 1,
      "summary": "测试新闻摘要",
      "content": "测试新闻内容",
      "date": "2025-01-18",
      "question": "测试题目",
      "answer": "测试答案"
    }
  ]'::jsonb
)
ON CONFLICT (date) DO NOTHING;

-- 查询测试数据
SELECT * FROM daily_news LIMIT 5;
```

### 1.3 点击 **Run** 执行SQL

如果看到类似这样的输出，说明表创建成功：
```
Create Table
Create Index
Create Trigger
```

## 第二步：获取API密钥（1分钟）

### 2.1 找到API设置
1. 点击左侧菜单的 **Project Settings**（齿轮图标）
2. 选择 **API** 选项卡

### 2.2 复制以下信息

#### ① Project URL
找到 **Project URL** 字段，格式如：
```
https://xxxxxxxxxxxxx.supabase.co
```
点击复制按钮，保存到记事本。

#### ② anon public key
找到 **anon public** 字段，格式如：
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```
点击复制按钮，保存到记事本。

### 2.3 安全提示
⚠️ **重要**：
- `anon` key 是公开密钥，可以安全地在客户端使用
- `service_role` key 是管理员密钥，**不要**提交到代码仓库
- 本项目只需要使用 `anon public` key

## 第三步：配置数据库权限（1分钟）

### 3.1 打开 Table Editor
1. 点击左侧菜单的 **Table Editor**
2. 找到 `daily_news` 表
3. 点击该表

### 3.2 检查表结构
确认表包含以下列：
- `id` (integer, primary key)
- `date` (text)
- `data` (jsonb)
- `created_at` (timestamp with time zone)
- `updated_at` (timestamp with time zone)

### 3.3 配置行级安全策略（RLS）- 可选

如果需要配置访问控制，在SQL Editor中执行：

```sql
-- 启用行级安全
ALTER TABLE daily_news ENABLE ROW LEVEL SECURITY;

-- 允许所有人读取数据
CREATE POLICY "Allow public read access"
ON daily_news FOR SELECT
TO anon
USING (true);

-- 允许所有人插入数据
CREATE POLICY "Allow public insert access"
ON daily_news FOR INSERT
TO anon
WITH CHECK (true);

-- 允许所有人更新数据
CREATE POLICY "Allow public update access"
ON daily_news FOR UPDATE
TO anon
USING (true);
```

**注意**：如果不想配置RLS，可以保持默认状态（public schema，无需权限）

## 第四步：测试连接（2分钟）

### 4.1 在项目本地测试

打开终端，执行：

```bash
cd /workspace/projects/server

# 创建临时测试文件
cat > test-supabase.js << 'EOF'
const { createClient } = require('@supabase/supabase-js');

const SUPABASE_URL = process.env.SUPABASE_URL || 'https://你的URL.supabase.co';
const SUPABASE_ANON_KEY = process.env.SUPABASE_ANON_KEY || '你的密钥';

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

async function testConnection() {
  console.log('🔗 测试Supabase连接...');
  console.log('URL:', SUPABASE_URL);
  console.log('Key:', SUPABASE_ANON_KEY.substring(0, 20) + '...');
  
  try {
    // 测试查询
    const { data, error } = await supabase
      .from('daily_news')
      .select('*')
      .limit(1);
    
    if (error) {
      console.error('❌ 连接失败:', error.message);
      process.exit(1);
    }
    
    console.log('✅ 连接成功！');
    console.log('📊 数据:', data);
  } catch (err) {
    console.error('❌ 错误:', err.message);
    process.exit(1);
  }
}

testConnection();
EOF

# 运行测试（替换为你的实际URL和密钥）
SUPABASE_URL="https://你的URL.supabase.co" \
SUPABASE_ANON_KEY="你的密钥" \
node test-supabase.js
```

### 4.2 预期输出
如果成功，会看到：
```
🔗 测试Supabase连接...
URL: https://你的URL.supabase.co
Key: eyJhbGciOiJIUzI1NiIs...
✅ 连接成功！
📊 数据: [ { id: 1, date: '2025-01-18', data: [...], ... } ]
```

## 第五步：配置Railway环境变量（1分钟）

### 5.1 登录Railway
访问 https://railway.app 并使用GitHub账号登录

### 5.2 创建新项目
1. 点击 **New Project**
2. 选择 **Deploy from GitHub repo**
3. 选择你的项目仓库

### 5.3 配置环境变量
在Railway项目页面：
1. 点击 **Variables** 标签
2. 添加以下变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `SUPABASE_URL` | 你的Supabase URL | 从第二步复制 |
| `SUPABASE_ANON_KEY` | 你的Supabase anon key | 从第二步复制 |
| `COZE_API_KEY` | pat_xxxxx | 从Coze获取 |
| `PORT` | 5000 | 固定值 |
| `NODE_ENV` | production | 固定值 |

### 5.4 重新部署
添加环境变量后，Railway会自动重新部署

## 常见问题

### Q1: SQL执行失败？
**A**: 检查SQL语法是否正确，确保没有多余空格。尝试逐条执行。

### Q2: 找不到API密钥？
**A**: 确保在 Project Settings → API 页面，不是在 Database Settings

### Q3: 连接测试失败？
**A**: 
- 检查URL和密钥是否正确复制（没有多余空格）
- 确认表已创建
- 检查网络连接

### Q4: Railway部署后无法连接？
**A**: 
- 确认环境变量已正确配置
- 等待Railway部署完成（查看Deploy日志）
- 使用curl测试：`curl https://你的railway域名/api/v1/health`
